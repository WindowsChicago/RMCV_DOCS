## Typora 通用破解教程

**by ZYL**

### 目前各版本破解情况：（截止2025年12月）

1.0以前的版本为免费版本

1.04/1.1.5/1.2.5 无图片./功能，旧UI，菜单栏点按为灰色特效，最完美破解只用执行替换html即可，不需要修改js伪激活，当然更推荐使用伪激活方法，虽然麻烦点但是跳过激活窗口比html方法更自然些

1.3.9/1.4.8/1.5.12开始有图片./功能但是还存在旧UI，菜单栏点按有红色特效，也是最完美破解，只用执行替换html即可，不需要修改js伪激活，当然更推荐使用伪激活方法，虽然麻烦点但是跳过激活窗口比html方法更自然些

1.6.7/1.7.6/1.8.10/1.95 新UI，仍然可以使用替换html的方法隐藏“未激活”字样和隐藏激活窗口，但是可能会有概率触发“a Java Script error”的报错，且出现报错后点击确定后程序会退出，需要重新打开，其中1.95触发的概率较低

1.10.8  可以使用替换html的方法隐藏”未激活“字样，但是不能使用html替换的方法隐藏激活窗口，否则即使改了注册表也可能会闪退，只能先使用js替换方法伪激活并使用js替换的方法隐藏激活窗口才行，并且可能还会有概率触发“a Java Script error”的报错，但是点击确定后不会退出程序，而是能正常运行，并且概率与1.9.5差不多

1.11.7/1.12.4 取消了试用期，可以无限试用，只是有”未激活“字样和激活弹窗，html替换方法完全失效，只能使用json替换方法隐藏”未激活“字样，并且js替换方法伪激活并隐藏激活窗口后会触发闪退，并且修改注册表也不行

**本教程的原理并不是破解激活机制，而是去除“未激活”字样和不显示激活窗口的伪激活或无限试用**

### Windows版激活方法：

#### **一.安装：**

先使用官方安装包安装Typora，安装完成后会自动启动一次，提示试用期xx天或试用期已过，关闭程序

#### **二.定位安装目录**

然后找到Typora的安装目录，如果安装时未手动修改一般默认在C:\Program Files\Typora

#### **三.隐藏左下角”未激活“字样：**

**方法1：**html替换方法：在Typora\resources文件夹找到window.html文件，使用记事本或者其他编辑器打开这个文件，在`<head>`后添加如下内容

```
<style>body>div[role="button"]{visibility:hidden;}</style>
```

保存修改，如果因为在C盘无法保存可以先另存到其他非C盘位置，然后在文件资源管理器里复制该修改后的文件，到原文件位置粘贴替换原文件

**方法2：**json替换方法：在Typora\resources\locales\zh-Hans.lproj文件夹下找到“Panel.json”文件，使用记事本或者其他编辑器打开这个文件，使用Ctrl+F进行查找：

```
"UNREGISTERED":"未激活"
```

将其替换为：

```
"UNREGISTERED":" "
```

注意“ ”之间是空格不是无字符，使用和3相同的方法保存修改，此时打开Typora就会发现“未激活”字样没了，但是”X“按钮还在

#### 四.伪激活&跳过激活窗口：

**方法1：**js替换方法：

伪激活：在Typora\resources\page-dist\static\js文件夹里寻找一个文件名类似“LicenseIndex.180dd4c7.bffb5802.chunk.js”（文件也可能不存在“chunk”字样）的文件，使用记事本或者其他编辑器打开这个文件，使用Ctrl+F进行查找：

```
e.hasActivated="true"==e.hasActivated
```

 将其替换为：

```
e.hasActivated="true"=="true"
```

保存修改，如果因为在C盘无法保存可以先另存到其他非C盘位置，然后在文件资源管理器里复制该修改后的文件，到原文件位置粘贴替换原文件

跳过激活窗口：在Typora\resources\page-dist\static\js下找到文件名类似”0.99879679.chunk.js“的文件，使用记事本或者其他编辑器打开这个文件，在最开始处添加一行，内容如下：

```
setTimeout(function() { document.querySelector('.default-btn.secondary-btn').click(); }, 32);
```

其中数字是显示多少ms关闭的数值，最小是32，低于32无效

**方法2：**html替换方法：（仅能用于1.9.5及更早的版本，1.10后所有修改html的方法都会触发程序自锁而无限闪退）

无须伪激活，在Typora\resources\page-dist文件夹下找到“license.html”，使用记事本或者其他编辑器打开这个文件，使用Ctrl+F进行查找：

```
</body></html>
```

替换为：

```
</body><script>window.οnlοad=function(){setTimeout(()=>{window.close();},5);}</script></html>
```

保存修改，如果因为在C盘无法保存可以先另存到其他非C盘位置，然后在文件资源管理器里复制该修改后的文件，到原文件位置粘贴替换原文件

#### 五.修复闪退：

此时打开Typora可能会发现程序闪退（尤其是高版本比如1.10），原因是因为触发了Typora自锁，此时需要修改注册表，win+r，输入regedit，在目录树中定位到“计算机\HKEY_CURRENT_USER\SOFTWARE\Typora”，在“Typora”处右键，选择“权限”，在弹出的窗口中选择“Administrators”，勾选两个“拒绝”，点击“确定”，这样即可正常打开程序了。

原理是在Windows系统里，给Typora的权限为Adminnistrators组勾选拒绝，意义是明确禁止了属于该组用户执行该权限对应的操作，且拒绝的权限高于允许。这里勾选后就Typora就没有权限读取和写入特定的注册表项了，这样可以绕过Typora的验证机制，防止软件自锁。因为Typora会将配置信息、许可证状态等数据存储在注册表里。
